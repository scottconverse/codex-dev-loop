# Runbook: Voice Feedback Loop

## Trigger

Use when the user gives dictated, rough, or transcript-style feedback.

## Steps

1. Save raw input in `inbox/` when useful.
2. Distill it into actions, decisions, preferences, questions, and open loops.
3. Create a goal card when implementation is implied.
4. Update memory files with durable facts.
5. Execute safe local changes.
6. Return a concise summary of what was understood, changed, and still needs judgment.
