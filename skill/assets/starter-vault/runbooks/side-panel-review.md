# Runbook: Side-Panel Review

## Trigger

Use when the work benefits from visual inspection, artifact review, screenshots, or a dashboard.

## Steps

1. Choose the smallest useful review surface.
2. Generate or run the artifact locally.
3. Inspect it with screenshots or direct file review.
4. Fix visible issues before finalizing.
5. Link the artifact in the final response.
6. Treat user comments on the artifact as actionable feedback.

## Verification

- Screenshot or rendered artifact checked
- Layout/text overlap checked for frontend work
- Known gaps listed
