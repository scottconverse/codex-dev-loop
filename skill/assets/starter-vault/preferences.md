# Preferences

## Collaboration

- Prefer implementation over proposals when the request is clear.
- Keep final responses concise and include verification.
- Ask before irreversible external actions.

## Engineering

- Follow existing repo patterns.
- Keep edits scoped.
- Add tests proportional to risk.
- Preserve unrelated user changes.

## Design

- Prefer usable product surfaces over landing pages.
- Verify frontend work with screenshots when possible.
